## 용어 사전 (뭐야 + 왜 나왔어 + 어원)

### Claude Code

| 항목 | 내용 |
|------|------|
| **뭐야** | 터미널에서 AI랑 대화하면서 코딩하는 도구 |
| **왜 나왔어** | ChatGPT로 코딩할 때 복붙이 너무 귀찮았음. "AI가 직접 파일을 수정하면 안 돼?" → Claude Code 탄생 |
| **어원** | **Claude** (클로드 섀넌, 정보이론의 아버지) + **Code** |
| **이전엔 어떻게 했어** | ChatGPT 답변 복사 → 에디터 붙여넣기 → 에러나면 다시 복사... 반복 |

```
[이전]
ChatGPT: "이렇게 수정하세요: ..."
개발자: (복사) (붙여넣기) (실행) "에러남"
ChatGPT: "그럼 이렇게..."
개발자: (복사) (붙여넣기)...

[지금]
Claude Code: 수정할까요? (y/n)
개발자: y
Claude Code: ✓ 완료
```

### Terminal (터미널)

| 항목 | 내용 |
|------|------|
| **뭐야** | 검은 화면에 글자만 나오는 프로그램 |
| **왜 나왔어** | 1960년대, 컴퓨터는 방 하나 크기였음. 여러 사람이 쓰려면 "단말기(terminal)"가 필요했음 |
| **어원** | 라틴어 **terminus** (끝) → 중앙 컴퓨터의 "끝단"에 있는 입출력 장치 |
| **이전엔 어떻게 했어** | 천공카드에 구멍 뚫어서 제출 → 다음날 결과 받음 |

```
[1960년대]
┌─────────────┐
│ 메인프레임   │ ← 방 하나 크기 컴퓨터
│ (중앙)      │
└──────┬──────┘
       │
   ┌───┴───┐
   ↓       ↓
[터미널] [터미널]  ← 여러 사람이 동시에 접속
```

**왜 아직도 써?**
- GUI보다 빠름 (마우스 클릭 vs 타이핑)
- 자동화 가능 (스크립트)
- 서버는 대부분 터미널만 있음

### curl

| 항목 | 내용 |
|------|------|
| **뭐야** | 인터넷에서 파일 다운로드하는 명령어 |
| **왜 나왔어** | 1998년, 브라우저 없이 터미널에서 HTTP 요청하고 싶었음. 서버 자동화할 때 브라우저 열 수 없잖아 |
| **어원** | **C**lient **URL** = 클라이언트가 URL에서 가져옴 |
| **만든 사람** | Daniel Stenberg (스웨덴, 취미 프로젝트로 시작) |
| **이전엔 어떻게 했어** | 브라우저 열고 → 다운로드 클릭 → 파일 옮기고... |

```bash
# 이게 없었으면?
1. 브라우저 열기
2. https://claude.ai/install.sh 접속
3. "다른 이름으로 저장"
4. 터미널에서 bash install.sh 실행

# curl 있으니까
curl -fsSL https://claude.ai/install.sh | bash
# 한 줄 끝
```

**옵션 해석:**
```
-f  fail silently (에러나면 조용히 종료)
-s  silent (진행바 숨김)
-S  show error (에러는 보여줌)
-L  location (리다이렉트 따라감)
```

### brew (Homebrew)

| 항목 | 내용 |
|------|------|
| **뭐야** | Mac용 패키지 관리자 |
| **왜 나왔어** | 2009년, Mac에서 개발 도구 설치가 너무 어려웠음. Linux는 `apt-get`이 있는데 Mac은 없었음 |
| **어원** | **Homebrew** = 집에서 만든 맥주. "Apple이 제공하지 않는 것을 우리가 양조한다" |
| **만든 사람** | Max Howell (아이러니하게도 나중에 Apple 면접 떨어짐) |
| **이전엔 어떻게 했어** | 소스코드 다운로드 → ./configure → make → make install → 의존성 지옥 |

```bash
# 이게 없었으면?
wget https://ftp.gnu.org/gnu/git/git-2.40.0.tar.gz
tar -xzf git-2.40.0.tar.gz
cd git-2.40.0
./configure
make
sudo make install
# + 의존성 libcurl, libssl 등 하나하나 설치...

# brew 있으니까
brew install git
# 끝
```

**왜 --cask가 따로 있어?**
```
Formula (레시피)     Cask (술통)
= 소스에서 빌드       = 이미 만들어진 앱
= CLI 도구           = GUI 앱
= git, node, curl    = Chrome, Slack, Claude Code
```

### npm

| 항목 | 내용 |
|------|------|
| **뭐야** | JavaScript 패키지 관리자 |
| **왜 나왔어** | 2010년, JS 라이브러리마다 사이트 가서 다운로드하기 귀찮았음. Python은 pip가 있는데 JS는 없었음 |
| **어원** | **N**ode **P**ackage **M**anager |
| **만든 사람** | Isaac Z. Schlueter |
| **이전엔 어떻게 했어** | jQuery.com 가서 다운로드 → js 폴더에 넣기 → script 태그 추가... 버전 관리? 수동으로 |

```bash
# 이게 없었으면?
1. jquery.com 접속
2. "Download" 클릭
3. jquery-3.6.0.min.js 저장
4. <script src="jquery-3.6.0.min.js"> 추가
5. 업데이트? 다시 1번부터...

# npm 있으니까
npm install jquery
# package.json에 자동 기록, 버전 관리됨
```

**-g는 뭐야?**
```
npm install express      # 이 프로젝트에만 설치
npm install -g typescript  # 컴퓨터 전체에 설치 (global)
```

### bun

| 항목 | 내용 |
|------|------|
| **뭐야** | npm/node보다 빠른 JavaScript 런타임 |
| **왜 나왔어** | 2022년, npm이 너무 느렸음. node_modules 설치하다 점심시간 끝남. "처음부터 다시 만들면?" |
| **어원** | **bun** = 빵 🥐 (창립자가 좋아함) + bundle(번들) 발음 비슷 |
| **만든 사람** | Jarred Sumner (전 Stripe 엔지니어) |
| **이전엔 어떻게 했어** | npm install → 3분 대기 → 커피 타러 감 |

```bash
# npm (느림)
$ npm install
added 1423 packages in 2m 34s

# bun (빠름)
$ bun install
+ 1423 packages in 4.2s  # 30배 빠름
```

### MCP

| 항목 | 내용 |
|------|------|
| **뭐야** | AI가 외부 도구를 쓸 수 있게 해주는 프로토콜 |
| **왜 나왔어** | AI는 "감옥"에 갇혀 있었음. 인터넷 검색도, 파일 읽기도, API 호출도 못함. "AI한테 손발을 달아주자" |
| **어원** | **M**odel **C**ontext **P**rotocol = 모델에게 컨텍스트를 제공하는 규약 |
| **만든 곳** | Anthropic (2024년, 오픈소스로 공개) |
| **이전엔 어떻게 했어** | 사람이 중간에서 복붙. "GitHub 이슈 내용 알려줘" → 브라우저 열고 복사 → ChatGPT에 붙여넣기 |

```
[이전]
AI: "GitHub 이슈 내용을 보여주세요"
인간: (브라우저 열기) (복사) (붙여넣기)
AI: "수정 코드입니다"
인간: (복사) (GitHub에 붙여넣기)

[지금 - MCP 연결 후]
AI: "GitHub 이슈 342번 확인할게요... 수정하고 PR 만들었습니다"
인간: (승인만 클릭)
```

### Path (경로)

| 항목 | 내용 |
|------|------|
| **뭐야** | 파일 위치를 나타내는 문자열 |
| **왜 나왔어** | 파일이 많아지면서 폴더(디렉토리) 개념 등장. "이 파일이 어디 있는지" 표현 필요 |
| **어원** | 영어 **path** = 길. 폴더를 따라가는 "길" |
| **이전엔 어떻게 했어** | 파일 이름만 있었음 (초기 DOS). 같은 이름 파일 불가능 |

```
/Users/kim/projects/app/src/index.ts
│     │    │        │   │   └── 파일
│     │    │        │   └── 폴더
│     │    │        └── 폴더
│     │    └── 폴더
│     └── 사용자명
└── 루트 (최상위)
```

**특수 기호:**
| 기호 | 의미 | 왜 이렇게? |
|------|------|-----------|
| `~` | 홈 폴더 | 유닉스 전통. 물결표가 "집"처럼 보여서? |
| `.` | 현재 폴더 | 점 = "여기" |
| `..` | 상위 폴더 | 점 두 개 = "위로" |
| `/` | 구분자 | 슬래시 (Windows는 `\`) |

### deprecated

| 항목 | 내용 |
|------|------|
| **뭐야** | "이제 안 씀, 쓰지 마" |
| **왜 나왔어** | 소프트웨어가 발전하면서 옛날 방식이 문제가 됨. 하지만 갑자기 삭제하면 기존 사용자가 곤란. "경고는 주되, 바로 삭제는 안 함" |
| **어원** | 라틴어 **deprecare** = 기도로 피하다 → "권장하지 않는다" |
| **과정** | deprecated 경고 → 몇 버전 후 removed |

```
v1.0: npm install 지원
v2.0: npm install (deprecated 경고 표시) ← 지금 여기
v3.0: npm install 완전 삭제 예정
```

### OAuth

| 항목 | 내용 |
|------|------|
| **뭐야** | 비밀번호 없이 로그인 권한 주는 방식 |
| **왜 나왔어** | 2007년, 앱에 비밀번호 직접 입력하면 위험했음. 앱이 해킹당하면 비번 유출. "비번 대신 임시 열쇠를 주자" |
| **어원** | **O**pen **Auth**orization = 개방형 인가 |
| **이전엔 어떻게 했어** | 앱에 아이디/비번 직접 입력 → 앱이 비번 저장 → 해킹되면 끝 |

```
[이전 - 위험]
앱: "비밀번호 입력하세요"
사용자: "mypassword123"
앱: (저장해둠) → 해킹당하면 비번 유출

[지금 - OAuth]
앱: "GitHub로 로그인"
→ GitHub 사이트로 이동
GitHub: "이 앱에 권한 줄까요?"
→ "허용" 클릭
→ 앱에 임시 토큰만 전달 (비번 아님)
→ 토큰은 언제든 취소 가능
```

### sudo

| 항목 | 내용 |
|------|------|
| **뭐야** | 관리자 권한으로 실행 |
| **왜 나왔어** | 보안. 일반 사용자가 시스템 파일 함부로 못 건드리게. "정말 필요할 때만 관리자 권한 사용" |
| **어원** | **S**uper**u**ser **do** = 슈퍼유저로 실행 |
| **이전엔 어떻게 했어** | 항상 관리자로 로그인 → 실수로 시스템 파일 삭제 → 컴퓨터 고장 |

```bash
# sudo의 철학
"필요할 때만, 최소한의 권한으로"

# ❌ 항상 관리자 (위험)
root$ rm -rf /  # 시스템 전체 삭제... 복구 불가

# ✅ 필요할 때만 sudo
user$ rm -rf /           # 권한 없음, 거부됨
user$ sudo rm -rf /tmp   # 특정 폴더만, 비번 확인 후 실행
```
